NAME

Math::GLPK::LP - Perl extension that allows to access many of GLPK's linear programming functions.

SYNOPSIS

  # create GLPK Simplex Environment
  my $glpk_env = Math::GLPK::EnvSimplex->new();

  # create GLPK linear program
  my $lp = $glpk_env->createLP();

  # define type of optimization
  $lp->maximize();

  # define columns
  my $lo = &Math::GLPK::Base::GLP_LO; #   lb < x < +inf Variable with lower bound
  my $cv = &Math::GLPK::Base::GLP_CV; # continuous variable
  my $cols = { num_cols  => 2,
               obj_coefs => [ 0.6,  0.5],
               types_bnd => [ $lo,  $lo],
               lower_bnd => [ 0.0,  0.0],
               upper_bnd => [ 0.0,  0.0],
               col_types => [ $cv,  $cv],
               col_names => ['c1', 'c2']};
  die "ERROR: newcols() failed\n" unless $lp->newcols($cols);

  # define rows
  # constraint matrix:
  #  1.0   2.0
  #  3.0   1.0
  my $up = &Math::GLPK::Base::GLP_UP; # -inf < x <   ub Variable with upper bound
  my $newRows;
  $newRows->[0][0] = 1.0;
  $newRows->[0][1] = 2.0;
  $newRows->[1][0] = 3.0;
  $newRows->[1][1] = 1.0;
  my $rows = {num_rows  => 2,
              upper_bnd => [1.0, 2.0],
              lower_bnd => [0.0, 0.0],
              sense     => [ $up, $up],
              row_names => ["row1", "row2"],
              row_coefs => $newRows};

  # solve linear program
  $lp->simplex();

  # get results
  my $status = $lp->get_status();
  my $obj_val = $lp->get_obj_val();

  for( my $c = 1; $c <= $lp->get_num_cols; $c++ )
  {
     print "column '", $lp->get_col_name($c), "': ", $lp->get_col_prim($c), "\n";
  }
  

  # free GLPL resources
  $lp->free();
  $glpk_env->close();

DESCRIPTION

Math::GLPK::LP is a parent class for Math::GLPK::LPSimplex, Math::GLPK::LPIpt, and Math::GLPK::LPMip. This set of classes is used to create, modify, solve, and free linear programs using GLPK's C library. Math::GLPK::LP and its children uses Math::GLPK::Base to communicate with the GLPK C library. GLPK is a free linear programming solvers that is available for most operating systems. Naturally, GLPK must be installed on your system if you want to use the module of Math::GLPK. The module was developed and tested with GLPK version 4.45.

EXPORT

None by default.

add_cols

Tell GLPK how many columns are going to be added to existing linear program.

  $lp->add_cols($num_cols);

add_rows

Tell GLPK how many rows are going to be added to existing linear program.

  $lp->add_rows($num_rows);

addrows

This method is a wrapper for a set of GLPK functions which need to be called when rows are added to the linear program.

  my $fr = &Math::GLPK::Base::GLP_FR; # -inf < x < +inf Free (unbounded) variable
  my $lo = &Math::GLPK::Base::GLP_LO; #   lb < x < +inf Variable with lower bound
  my $up = &Math::GLPK::Base::GLP_UP; # -inf < x <   ub Variable with upper bound
  my $db = &Math::GLPK::Base::GLP_DB; #   lb < x <   ub Double-bounded variable
  my $fx = &Math::GLPK::Base::GLP_FX; #   lb = x =   ub Fixed variable
  my $newRows;
  $newRows->[0][0] = 1.0;
  $newRows->[0][1] = 2.0;
  $newRows->[1][0] = 3.0;
  $newRows->[1][1] = 1.0;
  my $rows = {num_rows  => 2,
              upper_bnd => [1.0, 2.0],
              lower_bnd => [0.0, 0.0],
              sense     => [ $up, $up],
              row_names => ["row1", "row2"],
              row_coefs => $newRows};

  die "ERROR: addrows() failed\n" unless $lp->addrows($rows);

Instead of using addrows it is possible to execute the following methods to add new rows to the linear program: add_rows, set_row_name, set_row_bnds, and set_mat_row or load_matrix. Consequently, the above lines of code could be realized as follows:

  $num_rows  = 2,
  $ub        = [1.0, 2.0],
  $lb        = [0.0, 0.0],
  $sense     = [ $up, $up],
  $row_names = ["row1", "row2"],
  $newRows;
  $newRows->[0][0] = 1.0;
  $newRows->[0][1] = 2.0;
  $newRows->[1][0] = 3.0;
  $newRows->[1][1] = 1.0;

  $lp->add_rows($num_rows);
  for( my $r = 1; $r <= $num_rows; $r++ )
  {
     $lp->set_row_name($r, $row_names->[$r-1]);
     $lp->set_row_bnds($r, $sense->[$r-1], $lb->[$r-1], $ub->[$r-1]);
  }
  $lp->load_matrix($newRows);

adv_basis

This method constructs an advanced initial LP basis.

  $lp->adv_basis();

cpx_basis

This method constructs an initial basis with the algorithm proposed by R. Bixby.

  $lp->cpx_basis();

create_index

This routine creates the name index. The name index is an auxiliary data structure, which is intended to quickly (i.e. for logarithmic time) and rows and columns by their names. This routine can be called at any time. If the name index already exists, the routine does nothing.

  $lp->create_index();

This routine is supposed to be used together with find_row and find_col. e.g.

  $lp->create_index();
  print "row 'row1' found at index ",  $lp->find_row("row1"), "\n";
  print "row 'row2' found at index ",  $lp->find_row("row2"), "\n";
  print "column 'c1' found at index ", $lp->find_col("c1"), "\n";
  print "column 'c2' found at index ", $lp->find_col("c2"), "\n";
  $lp->delete_index();

del_cols

This routine deletes columns from the systems.

  # delete second and fourth column
  $lp->del_cols([2,4]);

  # delete third column
  $lp->del_cols([3]);

del_rows

This routine deletes rows from the systems.

  # delete first and third row
  $lp->delete_rows([1,3]);

  # delete second row
  $lp->delete_rows([2]);

delete_index

Delete name index.

  $lp->delete_index();

find_col

Find column by name. This method returns the name of the column which was set by set_col_name. If no column with the specified name was found, 0 is returned,

  $col_idx = $lp->find_col("c1");

find_row

Find row by name. This method returns the name of the row which was set by set_row_name. If no row with the specified name was found, 0 is returned,

  $row_idx = $lp->find_row("row1");

get_col_kind

This method returns the kind of a column.

  # return kind of second column
  $kind = $lp->get_col_kind(2)

The returned kind is either GLP_CV (continuous variable), GLP_IV (integer variable), or GLP_BV (binary variable).

get_col_lb

This method returns lower bound of a column.

  # get lower bound of first column
  $lb = $lp->get_col_lb(1);

get_col_name

This method returns name of a column.

  # get name of third column
  $name = $lp->get_col_name(3);

get_col_type

This method returns the type of a column.

  my $type = $lp->get_col_type(3);

The return type is either GLP_FR (free -unbounded- variable), GLP_LO (variable with lower bound), GLP_UP (variable with upper bound), GLP_DB (double-bounded variable), GLP_FX (fixed variable).

get_col_ub

This method returns upper bound of a column.

  # get upper bound of first column
  $ub = $lp->get_col_ub(1);

get_mat_col

This method retunrs (non-zero) elements of a column of the constraint matrix. The result is stored in a has where the key of a hash element is the row index.

  # get non-zero constraint values of the second column
  %nz_col_vals = $lp->get_mat_col(2);

get_mat_row

This method retunrs (non-zero) elements of a row of the constraint matrix. The result is stored in a has where the key of a hash element is the column index.

  # get non-zero constraint values of the third row
  %nz_row_vals = $lp->get_mat_row(3);

get_num_cols

Get number of columns of the linear program

   $num_col = $lp->get_num_cols();

get_num_nz

Get number of non-zero elements in constraint matrix of the linear program

  $num_nz = $lp->get_num_nz();

get_num_rows

Get number of rows of the linear program

   $num_rows = $lp->get_num_ros();

get_obj_coef

Get objective coefficient of a variable (column):

  # get objective coefficient of second variable/column
  $obj_col2 = $lp->get_obj_coef(2);

get_obj_dir

Get direction (minimize/maximize) of optimization

  $dir = $lp->get_obj_dir();

get_obj_name

Get name of objective function

  $obj_name = $lp->get_obj_name();

get_prob_name

Get name of linear problem

  $prob_name = $lp->get_prob_name();

get_rii

Get row scaling factor for a row

  # get scaling factor of third row
  my $scaling = $lp->get_rii(3);

get_row_lb

Get lower bound of a row

  # get lower bound of second row
  $lb = $lp->get_row_lb(2);

get_row_name

Get name of a row

  # get name of first row
  $name = $lp->get_row_name();

get_row_type

Get type of a row

  # get type of first row
  $type = $lp->get_row_type();

The return type is either GLP_FR (free -unbounded- variable), GLP_LO (variable with lower bound), GLP_UP (variable with upper bound), GLP_DB (double-bounded variable), GLP_FX (fixed variable).

get_row_ub

Get upper bound of a row

  # get upper bound of third row
  $lb = $lp->get_row_ub(3);

get_sjj

Get scaling factor of a column

  # get scaling factor of second column
  $scaling = $lp->get_sjj(2);

load_matrix

Load constraint matrix of a linear program

maximize

Tell GLPK to maximize the objective function

   $lp->maximize();

minimize

Tell GLPK to minimize the objective function

   $lp->minimize();

newcols

Define new columns of a linear problem

  $cv = &Math::GLPK::Base::GLP_CV; # continuous variable;
  $iv = &Math::GLPK::Base::GLP_IV; # integer variable;
  $bv = &Math::GLPK::Base::GLP_BV; # binary variable.

  $fr = &Math::GLPK::Base::GLP_FR; # -inf < x < +inf Free (unbounded) variable
  $lo = &Math::GLPK::Base::GLP_LO; #   lb < x < +inf Variable with lower bound
  $up = &Math::GLPK::Base::GLP_UP; # -inf < x <   ub Variable with upper bound
  $db = &Math::GLPK::Base::GLP_DB; #   lb < x <   ub Double-bounded variable
  $fx = &Math::GLPK::Base::GLP_FX; #   lb = x =   ub Fixed variable
  $cols = { num_cols  => 2,
            obj_coefs => [ 0.6,  0.5],
            types_bnd => [ $lo,  $lo],
            lower_bnd => [ 0.0,  0.0],
            upper_bnd => [ 0.0,  3.0],
            col_types => [ $iv,  $cv],
            col_names => ['c1', 'c2']};
  $lp->newcols($cols);

The method newcols is a wrapper fo Math::GLPK::LP::set_obj_coef, Math::GLPK::LP::et_col_name, Math::GLPK::LP::set_col_kind, and Math::GLPK::LP::set_col_bnds. Consequently, the above lines of code could be realized by:

  $cv = &Math::GLPK::Base::GLP_CV; # continuous variable;
  $iv = &Math::GLPK::Base::GLP_IV; # integer variable;
  $bv = &Math::GLPK::Base::GLP_BV; # binary variable.

  $fr = &Math::GLPK::Base::GLP_FR; # -inf < x < +inf Free (unbounded) variable
  $lo = &Math::GLPK::Base::GLP_LO; #   lb < x < +inf Variable with lower bound
  $up = &Math::GLPK::Base::GLP_UP; # -inf < x <   ub Variable with upper bound
  $db = &Math::GLPK::Base::GLP_DB; #   lb < x <   ub Double-bounded variable
  $fx = &Math::GLPK::Base::GLP_FX; #   lb = x =   ub Fixed variable

  $num_cols = 2;
  $obj_coefs = [ 0.6,  0.5];
  $types_bnd = [ $lo,  $lo],
  $lower_bnd = [ 0.0,  0.0],
  $upper_bnd = [ 0.0,  3.0],
  $col_types = [ $iv,  $cv],
  $col_names = ['c1', 'c2'];

  $lp->add_cols($num_cols);
  for(my $c = 1; $c <= $num_cols; $c++ )
  {
    $lp->set_obj_coef($c, $obj_coefs->[$c-1]);
    $lp->set_col_name($c, $col_names->[$c-1]);
    $lp->set_col_kind($c, $col_types->[$c-1]);
    $lp->set_col_bnds($c, $types_bnd->[$c-1], $lower_bnd->[$c-1], $upper_bnd->[$c-1]);
  }

read_lp

Read problem data in CPLEX LP format from a text file.

  $lp->read_lp("linprob.lp");

read_mps

Read problem data in MPS format from a text file.

  $lp->read_mps("linprob.mps");

read_prob

Read problem data in the GLPK LP/MIP format from a text file.

  $lp->read_prob("linprob.glpk");

Returns 0 if the operation was successful. Typically, the file to be read is written by write_prob.

read_sol

Read basic solution from a text file

  $lp->read_sol("solution.txt");

Returns 0 if the operation was successful. Typically, the file to be read is written by write_sol.

scale_prob

Perform automatic scaling of problem data. The following scaling options are available GLP_SF_GM (perform geometric mean scaling), GLP_SF_EQ (perform equilibration scaling), GLP_SF_2N (round scale factors to nearest power of two), and GLP_SF_SKIP (skip scaling, if the problem is well scaled). The above parameters can be combined by a boolean OR operation.

  $lp->scale_prob(&Math::GLPK::Base::GLP_SF_GM | &Math::GLPK::Base::GLP_SF_2N);

set_col_bnds

Set bounds of a variable/column

  # set bounds of third variable/column
  $lb = 0.5;
  $up = 9.0;
  $type = &Math::GLPK::Base::GLP_DB;
  $lp->set_col_bnds(3 $type, $lb, $ub);

set_col_kind

Set kind of a variable/column

  # set kind of first column to 'binary'
  $kind = &Math::GLPK::Base::GLP_BV;
  $lp->set_col_kind(1, $kind);

set_col_name

Set name of a variable/column

  # set name of fourth column
  $lp->set_col_name(4, "variable_4");

set_col_stat

Set the current status of a column. The following options are allowed: GLP_BS (make the column basic), GLP_NL (make the column non-basic), GLP_NU (make the column non-basic and set it to the upper bound - if the column is not double-bounded, this status is equivalent to GLP_NL (only in the case of this routine)), GLP_NF (the same as GLP_NL (only in the case of this routine)), GLP_NS (the same as GLP_NL (only in the case of this routine)).

  # set status of second column to 'basic - make the constraint inactive'
  $lp->set_col_stat(2, &Math::GLPK::Base::GLP_BS);

set_mat_col

Set elements of constraint matrix of a column

  # set matrix elements of third column
  $new_vals = [3.0, -1.3, 0.0, 2.5];
  $lp->set_mat_col(3, $new_vals);

set_mat_row

Set elements of constraint matrix of a row

  # set matrix elements of second row
  $new_vals = [1.0, -0.3, 1.0, 0.0];
  $lp->set_mat_row(2, $new_vals);

set_obj_coef

Set objective coefficient of a variable/column

  # set objective coefficient of third variable/column
  $lp->set_obj_coef(3, -1.5);

set_obj_name

Set name of objective function

  $lp->set_obj_name("Smart_objective");

set_prob_name

Set name of linear problem

  $lp->set_prob_name("Biomass_optimization");

set_rii

  Set scaling factor of a row.

  # set scaling factor of second row to 2.0
  $lp->set_rii(2, 2.0);

set_row_bnds

Set bounds of a row. The following bound types are supported &Math::GLPK::Base::GLP_FR, &Math::GLPK::Base::GLP_LO, &Math::GLPK::Base::GLP_UP, &Math::GLPK::Base::GLP_DB, &Math::GLPK::Base::GLP_FX.

  # set bounds of sixth row
  $db = &Math::GLPK::Base::GLP_DB;
  $lp->set_row_bnds(6, $db, -3.0, -1.0);

set_row_name

Set name of a row

  # set name of second row
  $lp->set_row_name("IamNumber2");

set_row_stat

Set status of a row. The following options are allowed: GLP_BS (make the row basic), GLP_NL (make the row non-basic), GLP_NU (make the row non-basic and set it to the upper bound - if the row is not double-bounded, this status is equivalent to GLP_NL (only in the case of this routine)), GLP_NF (the same as GLP_NL (only in the case of this routine)), GLP_NS (the same as GLP_NL (only in the case of this routine)).

  # set status of second row
  $lp->set_row_stat(2, &Math::GLPK::Base::GLP_BS);

set_sjj

  Set scaling factor of a column.

  # set scaling factor of third column to 0.5
  $lp->set_sjj(3, 0.5);

sort_matrix

Sort elements of the constraint matrix rebuilding its row and column linked lists. On exit from this method the constraint matrix is unchanged. However, elements in the row linked lists become ordered by ascending column indices, and the elements in the column linked lists become ordered by ascending row indices.

  $lp->sort_matrix

std_basis

This method constructs the "standard" (trivial) initial LP basis.

  $lp->std_basis();

unscale_prob

Perform unscaling of problem data

  $lp->unscale_prob();

write_lp

Write linear problem in CPLEX LP format to file

  $lp->write_lp("linprob.lp");

Returns 0 if the operation was successful.

write_mps

Write linear problem in MPS format to file

  $lp->write_mps("linear_problem.mps");

write_prob

Write linear problem in GLPK LP/MIP format to a text file.

  $lp->write_prob();

write_sol

Write basic solution to a text file

  $lp->write_sol("solution.txt");

Returns 0 if the operation was successful.
